# Pillog — 프로젝트 기획서

> **Pill + Log** : 건강기능식품 성분 분석 & 맞춤 영양 추천 SaaS

---

## 1. 서비스 개요

### 1-1. 핵심 컨셉

특정 브랜드에 종속되지 않는 **중립적인 건강기능식품 정보 플랫폼**으로, 소비자가 영양제의 성분을 쉽게 이해하고 자신에게 맞는 제품을 찾을 수 있도록 돕는 서비스입니다.

### 1-2. 타겟 고객

- 일반 소비자 (개인)
- 여러 영양제를 동시에 복용하는 사람
- 영양제 성분을 이해하고 싶지만 전문 용어가 어려운 사람

### 1-3. 해결하려는 문제

- **개인 맞춤 영양 추천** — 건강 상태, 생활습관에 따른 영양제 추천
- **성분 분석 / 안전성 관리** — 성분 해석, 상호작용 체크, 과잉 섭취 경고

## 2. 주요 기능 설계

### 2-1. MVP 핵심 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 제품 검색 & 성분 조회 | 바코드 스캔(GS1/QR) 또는 이름 검색으로 제품의 성분/기능성 정보 확인. OCR 실패 시 수동 입력 폴백 | P0 |
| 성분 비교 | 최대 4개 제품을 나란히 비교 (성분 함량, 1일 권장량 대비 비율, 가성비) | P0 |
| 성분 쉬운 해석 | 전문 용어를 일반인이 이해할 수 있는 말로 번역 | P0 |
| 1일 영양소 섭취량 대시보드 | 등록한 영양제 기준 총 섭취량 vs 권장량(RDI) 시각화 | P0 |
| 복용 기록 관리 | 내가 먹고 있는 영양제 등록 및 복용 기록 | P1 |
| 상호작용 체크 | 등록된 영양제 간 성분 상호작용/과잉 섭취 경고 (상한 섭취량 UL 기반) | P1 |
| 제품 즐겨찾기 | 비교 후 관심 제품 저장 및 위시리스트 관리 | P1 |
| 공유 기능 | 비교 결과/내 영양제 목록을 카카오톡/링크로 공유 (바이럴 성장) | P1 |
| 맞춤 추천 | 건강 설문 기반 영양제 추천 (규칙 기반) | P2 |

### 2-2. 고도화 기능 (Post-MVP)

| 기능 | 설명 |
|------|------|
| AI 성분 해석 | Claude/OpenAI API 연동, 성분표 사진 OCR + 자연어 해석 |
| 의약품 상호작용 | 복용 중인 의약품과 영양제 간 상호작용 분석 |
| 커뮤니티 리뷰 | 사용자 후기 및 평점 시스템 |
| 가격 비교 | 동일 성분 기준 가성비 비교 |
| 복용 리마인더 | 푸시 알림 기반 복용 시간 알림 |
| 오프라인 지원 | 바코드 스캔 결과 캐싱, 네트워크 없이도 기본 조회 가능 |

### 3-2. 스택 상세

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| 모바일 앱 | Expo (React Native) | 바코드 스캔, iOS/Android 동시 지원, OTA 업데이트 |
| 웹 | Next.js | SEO, 랜딩페이지, SSR/SSG |
| 공유 언어 | TypeScript | 프론트/백엔드 로직 코드 공유 |
| DB | Supabase (PostgreSQL) | 서버리스, RLS 기반 보안, 실시간 구독 |
| 인증 | Supabase Auth | 카카오/구글 소셜 로그인 |
| API 서버 | Vercel Edge Functions | 서버리스, 빠른 배포, 글로벌 엣지 |
| AI | Claude API / OpenAI API | 성분 해석, 자연어 상담, OCR |
| 데이터 수집 | GitHub Actions + Python | 식약처 공공 데이터 크론잡 수집 |
| 파일 저장 | Supabase Storage | 제품 이미지, 성분표 사진 |
| 검색 | Supabase pg_trgm + Full-text Search | 한글 초성 검색, 오타 보정(fuzzy matching) |
| 모니터링 | Sentry + Vercel Analytics | 에러 추적, 사용자 행동 분석 |

### 3-3. 모바일/웹 동시 개발 전략

- **Expo (React Native)** 로 모바일 앱 개발 → `expo-camera`로 바코드 스캔 구현
- **Next.js** 로 웹 앱 개발 → SEO 최적화된 랜딩 페이지 + 웹 앱
- 두 프로젝트가 TypeScript를 공유하므로 API 호출, 비즈니스 로직을 공통 패키지로 추출
- 초기 리소스가 부족하면 **Expo Router 웹 지원**으로 단일 코드베이스 운영도 가능
- **모노레포 전략:** Turborepo 기반, 공유 범위 — API 클라이언트, 타입 정의, 비즈니스 로직
- **MVP 우선순위:** 웹(Next.js) 우선 개발 → 모바일은 웹 검증 후 확장

### 3-4. 데이터 파이프라인 상세

- **수집 주기:** 매일 1회 (GitHub Actions cron)
- **성분명 정규화:** 동일 성분의 다른 표기 매핑 테이블 구축 (예: "비타민C" = "아스코르브산" = "L-아스코르빈산")
- **데이터 품질 검증:** 수집 데이터 누락/불일치 자동 감지 및 알림 (Slack/Discord 웹훅)
- **실패 처리:** 수집 실패 시 자동 재시도 (최대 3회), 실패 지속 시 알림

## 4. AI / 추천 엔진 로드맵

### Phase 1 — 규칙 기반 (MVP)

- 건강 고민별 필요 성분 매핑 테이블 수작업 구축
- 사용자 설문(나이, 성별, 생활습관, 건강 고민) → 매칭 로직
- AI 없이 영양학 가이드라인 기반으로 동작

### Phase 2 — LLM 연동 (고도화)

- Claude API / OpenAI API 연동
- 성분표 사진 OCR → LLM이 쉬운 말로 해석
- 자연어 기반 영양 상담 챗봇
- 성분 간 상호작용 분석 자동화

### Phase 3 — 개인화 (장기)

- 사용자 복용 기록 + 건강 데이터 기반 개인화 추천
- 그래프 DB(Neo4j Aura) 활용 성분 상호작용 모델링
- 협업 필터링 기반 유사 사용자 추천

## 7. 규제 및 법적 고려사항

- **"정보 제공/비교" 포지셔닝 필수** — "진단"이나 "처방"으로 오해받지 않도록 면책 문구 명시
- 건강기능식품에 관한 법률상 "질병의 예방 및 치료를 위한 의약품이 아님"을 사용자에게 명확히 안내
- 식약처 공공 데이터 이용 시 출처 표시 의무
- 개인 건강 정보 수집 시 개인정보보호법 준수 (최소 수집, 동의, 파기 등)

## 10. 경쟁 서비스 분석

| 서비스 | 특징 | Pillog 차별점 |
|--------|------|--------------|
| 필리 (Pilly) | 자사 제품 맞춤 구독 | 중립적 제품 비교 (브랜드 무관) |
| 모노랩스 | 설문 기반 구독형 영양제 | 성분 분석 + 안전성 체크 기능 |
| 화해 (화장품) | 성분 분석 앱의 선례 | 건기식 버전의 "화해" 포지셔닝 |
| iHerb | 글로벌 건기식 플랫폼 | 한국 규제 기반 성분 해석 + 상호작용 분석 |
| Examine.com | 근거 기반 성분 정보 | 한국어 + 모바일 친화 + 개인화 |
| 약학정보원 | 의약품 상호작용 DB | 건기식 특화 + 소비자 친화적 UX |

Pillog의 핵심 차별화 요소는 **중립성**과 **투명한 성분 해석**입니다. 특정 브랜드 판매가 아닌, 소비자 관점의 정보 제공에 집중합니다.
